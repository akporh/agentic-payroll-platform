  PAYROLL CALCULATION ENGINE                                                                                                                                                                                                                                                 
                                                                                                                                                                                                                                                                             
  1. Gross Pay Calculator                                                                                                                                                                                                                                                    
  - Where: backend/domain/payroll/salary.py::calculate_gross                                                                                                                                                                                                                 
  - Inputs: List of salary components (code, amount)                                                                                                                                                                                                                         
  - Outputs: Decimal gross pay sum                                                                                                                                                                                                                                           
  - Timing: Sync, pure function                                                                                                                                                                                                                                              
  - Deps: None                                                                                                                                                                                                                                                               
                                                                                                                                                                                                                                                                             
  2. PAYE Tax Calculation                                                                                                                                                                                                                                                    
  - Where: backend/domain/rules/paye.py::calculate_paye, calculate_paye_for_period                                                                                                                                                                                           
  - Inputs: Gross income, progressive tax brackets (lower_limit, upper_limit, rate)                                                                                                                                                                                          
  - Outputs: Decimal tax amount (2dp rounded)                                                                                                                                                                                                                                
  - Timing: Sync, pure function                                                                                                                                                                                                                                              
  - Deps: None (brackets injected)                                                                                                                                                                                                                                         
                                                                                                                                                                                                                                                                             
  3. Net Pay Calculation                                                                                                                                                                                                                                                   
  - Where: backend/domain/payroll/calculator.py::calculate_net_pay                                                                                                                                                                                                           
  - Inputs: Gross income, tax bands                                                                                                                                                                                                                                          
  - Outputs: {gross, paye, net} as Decimals
  - Timing: Sync, pure function                                                                                                                                                                                                                                              
  - Deps: paye.py                                                                                                                                                                                                                                                            
   
  4. Pension Contribution                                                                                                                                                                                                                                                    
  - Where: backend/domain/rules/pension.py::calculate_pension                                                                                                                                                                                                              
  - Inputs: Pensionable base, employee_rate, employer_rate                                                                                                                                                                                                                   
  - Outputs: Tuple (employee_contribution, employer_contribution)                                                                                                                                                                                                          
  - Timing: Sync, pure function                                                                                                                                                                                                                                              
  - Deps: None (rates injected from statutory_rule)                                                                                                                                                                                                                          
                                                                                                                                                                                                                                                                             
  5. NHF Calculation                                                                                                                                                                                                                                                         
  - Where: backend/domain/rules/nhf.py::calculate_nhf                                                                                                                                                                                                                      
  - Inputs: Basic salary, rate (default 2.5%)                                                                                                                                                                                                                                
  - Outputs: Decimal NHF amount                                                                                                                                                                                                                                            
  - Timing: Sync, pure function                                                                                                                                                                                                                                              
  - Deps: None                                                                                                                                                                                                                                                               
                                                                                                                                                                                                                                                                             
  6. Rent Relief                                                                                                                                                                                                                                                             
  - Where: backend/domain/rules/rent_relief.py::calculate_rent_relief, calculate_rent_relief_for_period
  - Inputs: Annual rent paid, rate, cap, annualization_factor                                                                                                                                                                                                                
  - Outputs: Period rent relief (Decimal)                    
  - Timing: Sync, pure function                                                                                                                                                                                                                                              
  - Deps: PeriodContext                                 
                                                                                                                                                                                                                                                                             
  7. Sequential Payroll Executor                                                                                                                                                                                                                                             
  - Where: backend/domain/payroll/sequential_executor.py::run_sequential_payroll
  - Inputs: primary_code, meta_json, accumulated_results, salary_components, ctx                                                                                                                                                                                             
  - Outputs: {component_code: Decimal} dict for all calculated values           
  - Timing: Sync                                                                                                                                                                                                                                                             
  - Deps: Handler registry (pension.py, paye.py, nhf.py, rent_relief.py, component_metadata)                                                                                                                                                                                 
  - Execution priority order: BASIC(10) → HOUSING(20) → TRANSPORT(30) → CONSOLIDATED_ALLOWANCE(40) → GROSS_PAY(100) → PENSION_EMPLOYEE(200) → RENT_RELIEF(250) → TAXABLE_INCOME(300) → PAYE(400) → NHF(410) → HEALTH_INSURANCE(420) → DEVELOPMENT_LEVY(430) →                
  LIFE_INSURANCE(440) → NET_PAY(500)                                                                                                                                                                                                                                         
                                                                                                                                                                                                                                                                             
  8. Single Employee Executor                                                                                                                                                                                                                                                
  - Where: backend/domain/payroll/executor.py::execute_single_employee_payroll                                                                                                                                                                                               
  - Inputs: employee_id, components, tax_bands, statutory_rule_id/version, payroll_rule_ids, inputs, component_metadata, context, contract dates                                                                                                                             
  - Outputs: {payroll_run_id, employee_id, rules_context_snapshot, payroll_result, error}                                                       
  - Timing: Sync                                                                                                                                                                                                                                                             
  - Deps: sequential_executor.py (primary path), legacy calculator.py (fallback)                                                                                                                                                                                             
  - Notes: Proration via contract_start/end; dual-path — sequential when metadata present, hardcoded fallback otherwise                                                                                                                                                      
                                                                                                                                                                                                                                                                             
  9. Payroll Rules Evaluator                                                                                                                                                                                                                                                 
  - Where: backend/domain/payroll/rule_evaluator.py::apply_payroll_rules                                                                                                                                                                                                     
  - Inputs: Salary components, payroll rules, employee inputs, client metadata                                                                                                                                                                                               
  - Outputs: (modified_salary_components, rule_trace)                         
  - Timing: Sync                                                                                                                                                                                                                                                             
  - Deps: rule_set / rule_set_item tables (via snapshot)
  - Calculation methods: unit_multiplier, daily_rate_deduction, fixed_amount                                                                                                                                                                                                 
                                                                                                                                                                                                                                                                             
  10. Result Builder                                                                                                                                                                                                                                                         
  - Where: backend/domain/payroll/result_builder.py::build_payroll_result                                                                                                                                                                                                    
  - Inputs: Components list, tax bands                                                                                                                                                                                                                                       
  - Outputs: {gross_components_jsonb, deductions_jsonb, net_pay, calculations_snapshot_json}                                                                                                                                                                                 
  - Timing: Sync                                                                            
  - Deps: calculator.py                                                                                                                                                                                                                                                      
                       
  11. Proration (Hire/Termination Factor)                                                                                                                                                                                                                                    
  - Where: backend/domain/payroll/period_context.py::compute_hire_termination_factor                                                                                                                                                                                         
  - Inputs: PeriodContext, contract_start, contract_end                                                                                                                                                                                                                      
  - Outputs: Decimal in [0, 1]                                                                                                                                                                                                                                               
  - Timing: Sync, pure function                                                                                                                                                                                                                                              
  - Deps: PeriodContext

  PERIOD / RUN LIFECYCLE MANAGEMENT                     
                                                                                                                                                                                                                                                                             
  12. Period Context
  - Where: backend/domain/payroll/period_context.py::build_period_context                                                                                                                                                                                                    
  - Inputs: period_start, period_end, period_type (MONTHLY|FORTNIGHTLY|CUSTOM), working_days_override
  - Outputs: Frozen PeriodContext dataclass (calendar_days, working_days, annualization_factor, period_fraction)
  - Timing: Sync, pure                                                                                                                                                                                                                                                       
  - Deps: None                                                                                                                                                                                                                                                               
                                                                                                                                                                                                                                                                             
  13. Payroll Run State Machine                                                                                                                                                                                                                                              
  - Where: backend/domain/payroll/state_machine.py, backend/domain/payroll/status.py
  - States: DRAFT → CALCULATING → PARTIAL|CALCULATED → APPROVED → LOCKED → PAID                                                                                                                                                                                              
  - Functions: can_transition(current, next), transition(current, next)        
  - Timing: Sync                                                                                                                                                                                                                                                             
  - Deps: DB triggers enforce same rules at schema level                                                                                                                                                                                                                     
                                                                                                                                                                                                                                                                             
  14. Batch Processor                                                                                                                                                                                                                                                        
  - Where: backend/domain/payroll/batch_processor.py::process_payroll_run
  - Inputs: payroll_run_id, employees list, tax_bands, statutory config, execution_mode (atomic|isolated)                                                                                                                                                                    
  - Outputs: {payroll_run_id, results, totals}                                                           
  - Timing: Sync (iterates per-employee synchronously)                                                                                                                                                                                                                       
  - Deps: executor.py                                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                             
  15. Pure Run Executor                                                                                                                                                                                                                                                      
  - Where: backend/domain/payroll/run_executor.py::execute_payroll_run_pure                                                                                                                                                                                                  
  - Inputs: payroll_run_id, workspace_id, employees, tax_bands, statutory config, performed_by, execution_mode
  - Outputs: {payroll_run_id, results, totals, audit_logs, events, rules_context_snapshot, pay_cycle_definition}                                                                                                                                                             
  - Timing: Sync, pure (no DB access)                                                                                                                                                                                                                                        
  - Deps: batch_processor.py, audit_events.py, snapshot.py                                                                                                                                                                                                                   
                                                                                                                                                                                                                                                                             
  16. Run Persister                                                                                                                                                                                                                                                          
  - Where: backend/application/payroll_run_persister.py 
  - Inputs: Pure run executor output                                                                                                                                                                                                                                         
  - Outputs: Persisted payroll_run + payroll_result rows; side-effects: audit_log, event_store rows
  - Timing: Sync, transactional                                                                                                                                                                                                                                              
  - Deps: DB session, payroll_run_repo.py, audit_log_repo.py, event_store_repo.py

  RULES ENGINE                                                                                                                                                                                                                                                               
                                                                                                                                                                                                                                                                             
  17. Statutory Rules (JSONB)                                                                                                                                                                                                                                                
  - Where: backend/infra/db/models/statutory_rule.py, table statutory_rule
  - Columns: statutory_rule_id, state (DRAFT/LIVE), version, rules_jsonb, tax_method, country_code, effective_from                                                                                                                                                           
  - Content: Pension rates, relief config, NHF rate, health insurance rates                                       
  - Temporal: effective_from enables date-based rule selection                                                                                                                                                                                                               
                                                                                                                                                                                                                                                                             
  18. Tax Bands                                                                                                                                                                                                                                                              
  - Where: Table tax_band, FK to statutory_rule                                                                                                                                                                                                                              
  - Columns: tax_band_id, statutory_rule_id, lower_limit, upper_limit, rate                                                                                                                                                                                                  
  - Used by: paye.py (injected via run executor)                           
                                                                                                                                                                                                                                                                             
  19. Payroll Rules (Legacy)                                                                                                                                                                                                                                                 
  - Where: backend/infra/db/models/payroll_rule.py, table payroll_rule                                                                                                                                                                                                       
  - Columns: rule_id, workspace_id, rule_name, rule_definition_json, rule_type (EARNING|DEDUCTION), is_active                                                                                                                                                                
  - Notes: Per-row format; superseded by rule_sets for new code                                              
                                                                                                                                                                                                                                                                             
  20. Rule Sets (Temporal Versioning)                                                                                                                                                                                                                                        
  - Where: Tables rule_set, rule_set_item; backend/infra/db/models/rule_set.py, rule_set_item.py                                                                                                                                                                             
  - rule_set: id, workspace_id, effective_from, created_at                                                                                                                                                                                                                   
  - rule_set_item: id, rule_set_id, rule_name, rule_definition_json, rule_type                                                                                                                                                                                               
  - Purpose: Historical rate resolution for cross-period inputs                                                                                                                                                                                                              
                                                        
  21. Component Metadata                                                                                                                                                                                                                                                     
  - Where: backend/infra/db/models/component_metadata.py, tables component_metadata, client_component_metadata
  - Columns: component_code, country_code, version, effective_from, is_active, calculation_method, execution_priority, metadata_json                                                                                                                                         
  - Supported calculation_methods: salary_component, sum_earnings, pension_rule, paye_rule, nhf_rule, health_insurance_flat, development_levy_flat, life_insurance_rule, net_formula, taxable_income, rent_relief
                                                                                                                                                                                                                                                                             
  22. Rules Context Snapshot                                                                                                                                                                                                                                                 
  - Where: backend/domain/rules/snapshot.py::build_rules_context_snapshot                                                                                                                                                                                                    
  - v1 (legacy): IDs only — {statutory_rule_id, statutory_version, payroll_rule_ids}                                                                                                                                                                                         
  - v2 (full-content): Embedded full rule state — statutory rule + tax bands + rule_set + historical_rule_sets
  - Purpose: Enables replay without DB access
