## Recommended Sprint Sequence

### Sprint 1 — Make the lifecycle completable end-to-end
Complete the authorisation chain from the UI so a run can go all the way from trigger to paid.

| Story | Notes |
|---|-------|
| P0-1 Approve/Lock/Paid UI | Backend fully wired; just add buttons |
| P0-2 Retry UI button | Same — backend exists |
| P0-4 Gate reconciliation to LOCKED/PAID | One-line state check in service |
| P0-5 Fix reconciliation 500 → 409 | Catch IntegrityError, return 409 |
| P2-3 Log retry to audit trail | Low effort; closes compliance gap |
| P3-6 ORDER BY lower_limit on tax bands | Low risk, high correctness value |

**Definition of done:** A bureau operator can trigger payroll, retry failures, approve, lock, mark paid, and reconcile — entirely from the web UI, with correct PAYE calculation in all cases.

---

### Sprint 2 — Enable disbursement
Make it possible to produce bank and regulatory output files from a completed run.

| Story | Notes |
|---|-------|
| P0-3 Net pay export endpoint + UI | Core disbursement deliverable |
| P1-4 PAYE remittance export | FIRS monthly obligation |
| P1-5 Pension schedule export | PENCOM monthly obligation |
| P1-1 Recalculate totals after retry | Required before reconciliation is trustworthy |

**Definition of done:** After a run is LOCKED, the bureau can download a bank EFT file, a PAYE remittance schedule, and a pension contribution schedule from the UI.
