  1. Full Payroll Run                                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                             
  Trigger: POST /payroll/run                                                                                                                                                                                                                                                 

  A. Request Validation & Data Loading (payroll.py:27)
  │                                                                                                                                                                                                                                                                          
  ├─ A1. Load workspace ──────────────────────────────── SELECT workspace                                                                                                                                                                                                    
  ├─ A2. Idempotency check ───────────────────────────── SELECT payroll_run WHERE idempotency_key → early return if found                                                                                                                                                    
  ├─ A3. Load employees ──────────────────────────────── SELECT employee JOIN employee_contract JOIN salary_definition WHERE status=ACTIVE                                                                                                                                   
  ├─ A4. Temporal statutory rule ─────────────────────── SELECT statutory_rule WHERE effective_from ≤ period_end ORDER BY effective_from DESC, version DESC                                                                                                                  
  ├─ A5. Extract statutory rates ─────────────────────── Parse rules_jsonb → pension_employee_rate, pension_employer_rate, nhf_rate, etc.                                                                                                                                    
  ├─ A6. Load tax bands ──────────────────────────────── SELECT tax_band WHERE statutory_rule_id                                                                                                                                                                             
  ├─ A7. Load pay cycle ──────────────────────────────── SELECT pay_cycle WHERE workspace_id AND is_active=TRUE                                                                                                                                                              
  ├─ A8. Build period context ────────────────────────── build_period_context() → PeriodContext (pure)                                                                                                                                                                       
  ├─ A9. Temporal rule set resolution ────────────────── SELECT rule_set WHERE effective_from ≤ period_end → SELECT rule_set_item                                                                                                                                            
  ├─ A10. Load component metadata ────────────────────── SELECT component_metadata WHERE country_code AND is_active ORDER BY execution_priority                                                                                                                              
  ├─ A11. Load client component overrides ────────────── SELECT client_component_metadata WHERE workspace_id                                                                                                                                                                 
  ├─ A12. Cross-period input prefetch ────────────────── SELECT rule_set per historical ref_date; build PeriodContext per month                                                                                                                                              
  ├─ A13. Build rules context snapshot (v2) ──────────── build_rules_context_snapshot() → embedded full rule state (pure)                                                                                                                                                    
  │                                                                                                                                                                                                                                                                          
  B. Load Unclaimed Inputs (payroll.py:361)                                                                                                                                                                                                                                  
  │                                                                                                                                                                                                                                                                          
  ├─ B1. SELECT payroll_input WHERE payroll_run_id IS NULL → dict keyed by employee_id
  │                                                                                                                                                                                                                                                                          
  C. Pure Domain Execution (payroll_run_service.py:96)  
  │                                                                                                                                                                                                                                                                          
  ├─ C1. execute_payroll_run_pure() ─ NO DB ACCESS      
  │   ├─ Transition: DRAFT → CALCULATING                                                                                                                                                                                                                                     
  │   │   └─ build_transition_audit(), build_transition_event() → list accumulation
  │   ├─ process_payroll_run() (batch_processor.py)                                                                                                                                                                                                                          
  │   │   └─ Per employee: execute_single_employee_payroll()                                                                                                                                                                                                                 
  │   │       ├─ apply_payroll_rules() → rule_evaluator.py (pure, returns modified_components + rule_trace)                                                                                                                                                                  
  │   │       │   ├─ unit_multiplier: rate × quantity, rate resolved from historical rule set by reference_date                                                                                                                                                              
  │   │       │   ├─ daily_rate_deduction: (gross / working_days) × absent_days                                                                                                                                                                                              
  │   │       │   └─ fixed_amount: conditional flat addition                                                                                                                                                                                                                 
  │   │       ├─ run_sequential_payroll() → sequential_executor.py                                                                                                                                                                                                           
  │   │       │   └─ Dispatch through _HANDLERS by execution_priority:                                                                                                                                                                                                       
  │   │       │       10 BASIC → 20 HOUSING → 30 TRANSPORT → 40 CONSOLIDATED_ALLOWANCE                                                                                                                                                                                       
  │   │       │       → 100 GROSS_PAY → 200 PENSION_EMPLOYEE → 250 RENT_RELIEF                                                                                                                                                                                               
  │   │       │       → 300 TAXABLE_INCOME → 400 PAYE → 410 NHF                                                                                                                                                                                                              
  │   │       │       → 420 HEALTH_INSURANCE → 430 DEVELOPMENT_LEVY → 440 LIFE_INSURANCE                                                                                                                                                                                     
  │   │       │       → 500 NET_PAY                                                                                                                                                                                                                                          
  │   │       └─ build_payroll_result() → {gross_components_jsonb, deductions_jsonb, net_pay, calculations_snapshot_json}                                                                                                                                                    
  │   └─ Transition: CALCULATING → CALCULATED (or PARTIAL if failure_count > 0)                                                                                                                                                                                              
  │       └─ build_transition_audit(), build_transition_event()                                                                                                                                                                                                              
  │                                                                                                                                                                                                                                                                          
  D. Persist (payroll_run_persister.py) — SINGLE TRANSACTION                                                                                                                                                                                                                 
  │                                                                                                                                                                                                                                                                          
  ├─ INSERT payroll_run (status, rules_context_snapshot, period, totals, idempotency_key, ...)
  ├─ INSERT payroll_result ×N (per employee, with gross_components_jsonb, deductions_jsonb, net_pay, calculations_snapshot_json)                                                                                                                                             
  ├─ INSERT audit_log ×2 (DRAFT→CALCULATING, CALCULATING→CALCULATED/PARTIAL)                                                                                                                                                                                                 
  └─ INSERT event_store ×2                                                                                                                                                                                                                                                   
  │                                                                                                                                                                                                                                                                          
  E. Post-Persistence Input Claiming (payroll.py:531)   
  │                                                                                                                                                                                                                                                                          
  └─ UPDATE payroll_input SET payroll_run_id WHERE payroll_run_id IS NULL AND (reference_date IS NULL OR reference_date ≤ period_end)
                                                                                                                                                                                                                                                                             
  State transition: DRAFT → CALCULATING → CALCULATED | PARTIAL                                                                                                                                                                                                               
                                                                                                                                                                                                                                                                             
  Output: {status, payroll_run_id, totals: {total_gross_pay, total_deductions, total_net_pay, success_count, failure_count}}

  2. Retry Flow                                         

  Trigger: POST /payroll/retry

  R1. SELECT payroll_run FOR UPDATE (lock row)
  R2. Validate: must be PARTIAL; reject if PAID/APPROVED/LOCKED                                                                                                                                                                                                              
  R3. Branch on retry_strategy:                                                                                                                                                                                                                                              
  │                                                                                                                                                                                                                                                                          
  ├─ PER_EMPLOYEE:                                                                                                                                                                                                                                                           
  │   ├─ SELECT payroll_result WHERE status=FAILED AND payroll_run_id                                                                                                                                                                                                        
  │   ├─ If no FAILED rows → return early (idempotent)                                                                                                                                                                                                                       
  │   ├─ Rebuild context from payroll_run snapshot + live DB queries                                                                                                                                                                                                         
  │   └─ Per FAILED employee:                                                                                                                                                                                                                                                
  │       ├─ SELECT employee_contract, salary_definition                                                                                                                                                                                                                     
  │       ├─ execute_single_employee_payroll() (pure)                                                                                                                                                                                                                        
  │       ├─ DELETE payroll_result WHERE payroll_run_id AND employee_id AND status=FAILED                                                                                                                                                                                    
  │       └─ INSERT payroll_result (SUCCESS or FAILED)  
  │                                                                                                                                                                                                                                                                          
  └─ FULL_RUN:                                          
      ├─ DELETE ALL payroll_result WHERE payroll_run_id                                                                                                                                                                                                                      
      ├─ SELECT all employees for workspace                                                                                                                                                                                                                                  
      ├─ execute_single_employee_payroll() ×N (pure)                                                                                                                                                                                                                         
      └─ INSERT payroll_result ×N                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                                             
  R6. SELECT COUNT(*) payroll_result WHERE status=FAILED
  R7. UPDATE payroll_run SET status = (CALCULATED if 0 remain, else PARTIAL)                                                                                                                                                                                                 
  R8. COMMIT                                                                                                                                                                                                                                                                 
  
  State transition: PARTIAL → CALCULATED | PARTIAL                                                                                                                                                                                                                           
                                                        
  Gaps:
  - payroll_run.total_gross_pay / total_deduction / total_net_pay NOT recalculated after retry — stale totals
  - Retry operation itself not written to audit_log or event_store                                                                                                                                                                                                           
  - component_trace_jsonb on retried results: only populated if sequential executor runs
                                                                                                                                                                                                                                                                             
  ---                                                                                                                                                                                                                                                                        
  3. Approval / Lock / Paid Flow                                                                                                                                                                                                                                             
                                                                                                                                                                                                                                                                             
  Trigger: POST /payroll/approve, /lock, /mark-paid     
                                                                                                                                                                                                                                                                             
  Each transition follows same pattern:                 
  │                                                                                                                                                                                                                                                                          
  ├─ SELECT payroll_run FOR UPDATE (row-level lock)
  ├─ state_machine.transition(current, target) — raises ValueError on invalid                                                                                                                                                                                                
  ├─ UPDATE payroll_run SET status = target                                                                                                                                                                                                                                  
  ├─ COMMIT
  ├─ INSERT audit_log (entity_type, entity_id, action, old_value, new_value, performed_by)                                                                                                                                                                                   
  └─ INSERT event_store (type=PAYROLL_RUN_STATUS_CHANGED, payload={from, to})                                                                                                                                                                                                
                                                                                                                                                                                                                                                                             
  After PAID: DB trigger trg_prevent_paid_run_update blocks all further UPDATEs                                                                                                                                                                                              
                                                                                                                                                                                                                                                                             
  State transitions: CALCULATED → APPROVED → LOCKED → PAID                                                                                                                                                                                                                   
                                                        
  Gaps:                                                                                                                                                                                                                                                                      
  - No pre-condition guards — approve/lock/pay succeeds regardless of downstream completeness (e.g., can approve a PARTIAL run after it transitions to CALCULATED)
  - No bulk transition; each step requires a separate API call                                                                                                                                                                                                               
                                                        
  ---                                                                                                                                                                                                                                                                        
  4. Reconciliation Flow                                
                                                                                                                                                                                                                                                                             
  Trigger: POST /payroll/reconcile                      

  RC1. SELECT payroll_run.total_net_pay WHERE payroll_run_id
  RC2. Decimal comparison: actual_total vs expected_total → status = MATCHED | MISMATCH                                                                                                                                                                                      
  RC3. INSERT payroll_reconciliation (payroll_run_id, expected_total, actual_total, status, reconciled_at)                                                                                                                                                                   
  RC4. Return record                                                                                                                                                                                                                                                         
                                                                                                                                                                                                                                                                             
  Gaps:                                                                                                                                                                                                                                                                      
  - No run state validation — can reconcile a DRAFT or CALCULATING run                                                                                                                                                                                                       
  - No duplicate prevention — second reconcile call causes IntegrityError → 500 (not 409)
  - MISMATCH is recorded but no follow-up action triggered                               
                                                                                                                                                                                                                                                                             
  ---                                                                                                                                                                                                                                                                        
  5. Onboarding Flow                                                                                                                                                                                                                                                         
                                                                                                                                                                                                                                                                             
  Trigger: POST /onboarding/upload (preview only) or POST /onboarding/commit (writes)
                                                                                                                                                                                                                                                                             
  PREVIEW path (upload):
  │                                                                                                                                                                                                                                                                          
  ├─ emit_onboarding_sql() — pure, no DB writes         
  ├─ review_client_onboarding() — hard_validator + ai_critic + compliance (pure)                                                                                                                                                                                             
  ├─ If hard validation FAIL → return BLOCKED with errors                                                                                                                                                                                                                    
  └─ Return: {status, review, sql_statements}                                                                                                                                                                                                                                
                                                                                                                                                                                                                                                                             
  COMMIT path:                                                                                                                                                                                                                                                               
  │                                                                                                                                                                                                                                                                          
  ├─ review_client_onboarding() (re-validate)                                                                                                                                                                                                                                
  ├─ If FAIL → return error                             
  ├─ SELECT workspace (verify exists)
  ├─ ATOMIC TRANSACTION:                                                                                                                                                                                                                                                     
  │   ├─ INSERT OR UPDATE salary_definition ×N (ON CONFLICT)                                                                                                                                                                                                                 
  │   ├─ INSERT payroll_rule ×N                                                                                                                                                                                                                                              
  │   ├─ INSERT pay_cycle (ON CONFLICT DO NOTHING)                                                                                                                                                                                                                           
  │   ├─ INSERT grade ×N (ON CONFLICT DO NOTHING)                                                                                                                                                                                                                            
  │   ├─ INSERT designation ×N (ON CONFLICT DO NOTHING)                                                                                                                                                                                                                      
  │   ├─ INSERT OR UPDATE client_component_metadata ×N
  │   ├─ INSERT employee ×N                                                                                                                                                                                                                                                  
  │   ├─ INSERT employee_contract ×N (with resolved salary_def_id, grade_id, designation_id)
  │   ├─ INSERT rule_set (one per unique effective_from date)                                                                                                                                                                                                                
  │   └─ INSERT rule_set_item ×N                        
  ├─ COMMIT                                                                                                                                                                                                                                                                  
  └─ Auto-advance workspace state machine:              
      DRAFT → STRUCTURE_DEFINED → COMPENSATION_DEFINED → RULES_DEFINED → READY                                                                                                                                                                                               
      (each transition attempted in sequence; halts at first failing pre-condition)                                                                                                                                                                                          
                                                                                                                                                                                                                                                                             
  Gaps:                                                                                                                                                                                                                                                                      
  - No idempotency on commit — re-running same payload hits constraint violations (partial protection from ON CONFLICT clauses only)                                                                                                                                         
  - Grade/designation derivation from salary_def code silently leaves NULLs on failure                                                                                                                                                                                       
  - No cross-validation: employee salary_definition_id resolution is best-effort

  7. Payroll Input Flow                                                                                                                                                                                                                                                      
                                                        
  Trigger: POST /{workspace_id}/payroll/inputs (single or bulk)
                                                                                                                                                                                                                                                                             
  Single:
  │                                                                                                                                                                                                                                                                          
  ├─ SELECT payroll_rule WHERE workspace_id AND is_active → derive valid input_codes
  ├─ Validate input_code against codes dict                                                                                                                                                                                                                                  
  ├─ Parse reference_date → normalized to first day of month
  └─ INSERT payroll_input (payroll_run_id=NULL, employee_id, input_code, input_category, quantity, reference_date)                                                                                                                                                           
                                                                                                                                                                                                                                                                             
  Bulk:                                                                                                                                                                                                                                                                      
  │                                                                                                                                                                                                                                                                          
  ├─ SELECT payroll_rule (same code map)                
  ├─ SELECT employee WHERE workspace_id AND status=ACTIVE → employee_number → employee_id map                                                                                                                                                                                
  └─ Per row:                                                                                                                                                                                                                                                                
      ├─ Resolve employee_number → employee_id (skip with error if not found)                                                                                                                                                                                                
      ├─ Validate input_code                                                                                                                                                                                                                                                 
      └─ INSERT payroll_input                                                                                                                                                                                                                                                
                                                                                                                                                                                                                                                                             
  State: payroll_input.payroll_run_id = NULL (unclaimed) until claimed during payroll run execution (Step E above)                                                                                                                                                           
  
  Gaps:                                                                                                                                                                                                                                                                      
  - No quantity validation (negative, zero, or implausibly large values accepted)
  - No deduplication constraint — identical inputs can be inserted multiple times                                                                                                                                                                                            
  - Bulk import: employee resolution failure skips row silently, no batch rollback

  8. Rule Evaluation Subflow
                                                                                                                                                                                                                                                                             
  Called from: executor.py during payroll run (Step C above)
                                                                                                                                                                                                                                                                             
  apply_payroll_rules(salary_components, payroll_rules, employee_inputs, client_metadata)
  │                                                                                                                                                                                                                                                                          
  ├─ Normalize inputs: _to_events() converts scalars → [{quantity, reference_date}]
  ├─ Per rule (active only):                                                                                                                                                                                                                                                 
  │   ├─ unit_multiplier:                                                                                                                                                                                                                                                    
  │   │   └─ Per event:                                                                                                                                                                                                                                                      
  │   │       ├─ Resolve rate from historical_rule_sets by reference_date (fallback: current rate)                                                                                                                                                                           
  │   │       ├─ Resolve PeriodContext for event month (from historical_period_contexts)                                                                                                                                                                                     
  │   │       └─ amount = rate × quantity                                                                                                                                                                                                                                    
  │   ├─ daily_rate_deduction:                                                                                                                                                                                                                                               
  │   │   ├─ base = gross / working_days (from PeriodContext)                                                                                                                                                                                                                
  │   │   └─ deduction = base × absent_days                                                                                                                                                                                                                                  
  │   └─ fixed_amount:                                  
  │       ├─ Evaluate eligibility condition (input_present, input_value, component_result checks)                                                                                                                                                                            
  │       └─ If eligible: add fixed_amount to salary_components                                                                                                                                                                                                              
  ├─ Accumulate modified salary_components                                                                                                                                                                                                                                   
  └─ Return (modified_components, rule_trace)                                                                                                                                                                                                                                
                                                        
  Pure function — no DB access. All data injected as parameters.                                                                                                                                                                                                             
                                                        
  Gaps:                                                                                                                                                                                                                                                                      
  - Rate resolution failure is silent — if no historical rule set found, current rate used without warning
  - daily_rate_deduction has no bounds check — absent_days > working_days produces negative deduction without error                                                                                                                                                          
  - Eligibility condition evaluation references missing input_codes as False (silent)

  1. Full Payroll Run                                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                                           
  Execution logic produces:                                                                                                                                                                                                                                                  
  - Per-employee records in payroll_result: gross_components_jsonb, deductions_jsonb, net_pay, calculations_snapshot_json, component_trace_jsonb

  Outcome: Payroll calculations for all active employees in a workspace for a specific period are computed and persisted.                                                                                                                                                    
                                                                                                                                                                                                                                                                             
  Exposed via: GET /payroll/results/{workspace_id}/{run_id} — returns per-employee net pay, gross components, deductions.                                                                                                                                                    
                                                                                                                                                                                                                                                                             
  Classification: Admin/internal                                                                                                                                                                                                                                             
                                                                                                                                                                                                                                                                             
  ▎ No employee-facing endpoint exists. Results are readable by operators only. There is no payslip endpoint, no disbursement trigger, and no notification mechanism.                                                                                                        
                                                                                                                                                                                                                                                                           
  ---                                                                                                                                                                                                                                                                        
  2. Retry Flow                   
                                                                                                                                                                                                                                                                             
  Execution logic produces:                             
  - payroll_result rows: old FAILED rows deleted, new rows inserted (SUCCESS or FAILED)
  - payroll_run.status updated (PARTIAL → CALCULATED if all resolved)                  
  - Stale payroll_run.total_* aggregates (not recalculated)                                                                                                                                                                                                                
                                                                                                                                                                                                                                                                             
  Outcome: Previously failed employee calculations are reattempted without re-running the full batch.                                                                                                                                                                      
                                                                                                                                                                                                                                                                             
  Exposed via: POST /payroll/retry                                                                                                                                                                                                                                           
                                                                                                                                                                                                                                                                             
  Classification: Admin/internal — Incomplete                                                                                                                                                                                                                                
                                                                                                                                                                                                                                                                             
  ▎ Incomplete because total_gross_pay, total_deduction, total_net_pay on the run are NOT updated after retry. Any downstream consumer of run-level totals (e.g. reconciliation) will see stale figures. Retry is also not audited — no audit_log or event_store entry is    
  written for the retry operation itself.                                                                                                                                                                                                                                  
                                                                                                                                                                                                                                                                             
  ---                                                   
  3. Approve → Lock → Paid                                                                                                                                                                                                                                                   
                                                        
  Execution logic produces:
  - payroll_run.status transitions: CALCULATED → APPROVED → LOCKED → PAID
  - Two DB writes per transition: audit_log INSERT + event_store INSERT  
  - After PAID: DB trigger trg_prevent_paid_run_update makes run immutable at schema level                                                                                                                                                                                 
                                                                                                                                                                                                                                                                             
  Outcome: Payroll run is moved through an authorisation chain. Once PAID, the calculation record is permanently frozen.                                                                                                                                                   
                                                                                                                                                                                                                                                                             
  Exposed via: POST /payroll/approve, /lock, /mark-paid                                                                                                                                                                                                                      
                                                                                                                                                                                                                                                                             
  Classification: Admin/internal — Incomplete                                                                                                                                                                                                                                
                                                                                                                                                                                                                                                                             
  ▎ Incomplete because PAID is a terminal flag only — it does not trigger disbursement, does not generate a pay instruction file, does not notify any external system, and does not produce a payslip for employees. The PAID state signals "approved for payment" but the   
  actual payment mechanism is absent.

  4. Reconciliation                                                                                                                                                                                                                                                          
                                                        
  Execution logic produces:
  - payroll_reconciliation row: expected_total (from payroll_run.total_net_pay), actual_total (submitted externally), status (MATCHED or MISMATCH), reconciled_at
                                                                                                                                                                 
  Outcome: A manual comparison between the calculated net pay total and the amount actually disbursed is recorded.                                                                                                                                                         
                                                                                                                                                                                                                                                                             
  Exposed via: POST /payroll/reconcile, readable via GET /payroll/reconciliation/{run_id} (if wired)                                                                                                                                                                       
                                                                                                                                                                                                                                                                             
  Classification: Admin/internal — Incomplete                                                                                                                                                                                                                                
                                                                                                                                                                                                                                                                             
  ▎ Incomplete on two counts: (1) MISMATCH status is persisted but triggers no follow-up action — no alert, no block on future runs, no correction flow. (2) The expected_total used for comparison is the stale run-level total, which is not recalculated after retry (see 
  Gap G1). A reconciliation after a retry will compare against incorrect figures.

  5. Onboarding                                                                                                                                                                                                                                                              
                                                        
  Execution logic produces:
  - Workspace seeded with: salary_definition ×N, payroll_rule ×N, pay_cycle, grade ×N, designation ×N, employee ×N, employee_contract ×N, rule_set + rule_set_item ×N, client_component_metadata ×N
  - workspace.status auto-advanced up to READY                                                                                                                                                     
                                                                                                                                                                                                                                                                           
  Outcome: A workspace is fully configured and ready to execute payroll. Employees, compensation structures, and calculation rules are loaded in a single atomic operation.                                                                                                  
                                                                                                                                                                                                                                                                           
  Exposed via: POST /onboarding/upload (preview + SQL), POST /onboarding/commit (write)                                                                                                                                                                                      
                                                                                                                                                                                                                                                                             
  Classification: Admin/internal                                                                                                                                                                                                                                             
                                                                                                                                                                                                                                                                             
  ▎ Fully realizable. The READY state is a pre-condition checked before payroll runs. Preview path (upload) is non-destructive and returns generated SQL for review.

  6. Workspace Setup              
                                                                                                                                                                                                                                                                             
  Execution logic produces:                             
  - workspace row: workspace_id, name, country_code, base_currency, status=DRAFT
                                                                                
  Outcome: An empty workspace container is created as the root entity for a payroll bureau client.                                                                                                                                                                         
                                                                                                                                                                                                                                                                             
  Exposed via: POST /workspace                                                                                                                                                                                                                                             
                                                                                                                                                                                                                                                                             
  Classification: Admin/internal                        
                                                                                                                                                                                                                                                                             
  ▎ Realizable as a standalone step. However, a workspace in DRAFT state cannot execute payroll — it must progress through onboarding before it becomes functional.

  7. Payroll Input                
                                                                                                                                                                                                                                                                             
  Execution logic produces:                             
  - payroll_input rows with payroll_run_id=NULL (unclaimed)
  - Claimed at run time by link_inputs_to_run() (UPDATE payroll_input SET payroll_run_id)
                                                                                                                                                                                                                                                                           
  Outcome: Variable-period earnings or deductions (overtime hours, absent days, bonuses) are recorded against an employee and period, to be picked up by the next payroll run.                                                                                               
                                                                                                                                                                                                                                                                           
  Exposed via: POST /{workspace_id}/payroll/inputs, bulk variant, GET list, DELETE                                                                                                                                                                                           
                                                                                                                                                                                                                                                                             
  Classification: Admin/internal                                                                                                                                                                                                                                             
                                                                                                                                                                                                                                                                             
  ▎ Fully realizable within the payroll pipeline. Inputs without a reference_date are claimed by the next run regardless of period. Inputs with a reference_date are only claimed if reference_date ≤ period_end, enabling cross-period staging.

  8. Rule Evaluation (Subflow)                          
                                                                                                                                                                                                                                                                             
  Execution logic produces:                             
  - Modified salary_components dict (adjustments applied to base components)
  - rule_trace list (per-rule: method, rate used, resolution source, calculated amount)
                                                                                                                                                                                                                                                                           
  Outcome: Workspace-defined earning and deduction rules (overtime, shift allowance, absence deductions) are applied to an employee's base salary components before statutory calculations run.                                                                              
                                                                                                                                                                                                                                                                           
  Exposed via: Internal only — called inside executor.py, not directly accessible                                                                                                                                                                                            
                                                                                                                                                                                                                                                                             
  Classification: Backend-only                                                                                                                                                                                                                                               
                                                                                                                                                                                                                                                                             
  ▎ The rule_trace is embedded in calculations_snapshot_json within payroll_result, so it is accessible via the results endpoint. However, there is no standalone endpoint to test rule evaluation against sample inputs.

  9. Exports                      
                                                                                                                                                                                                                                                                             
  Execution logic produces:                             
  - net_pay_export.py → CSV: employee_id, net_pay per employee
  - paye_export.py → PAYE summary for government returns      
  - payroll_register_export.py → full payroll register (all components per employee)                                                                                                                                                                                       
                                                                                                                                                                                                                                                                             
  Outcome: Structured output files for disbursement, tax filing, and audit.                                                                                                                                                                                                
                                                                                                                                                                                                                                                                             
  Exposed via: Not exposed — no API endpoint wires any export function. Scripts in backend/scripts/ call them directly.                                                                                                                                                      
                                                                                                                                                                                                                                                                             
  Classification: Backend-only — Incomplete                                                                                                                                                                                                                                  
                                                                                                                                                                                                                                                                             
  ▎ Export logic exists but has no callable entry point beyond manual script execution. No HTTP endpoint, no triggered generation on run completion, no file storage. Effectively unreachable in a deployed context.
