## Grade Decoupling ‚Äî Bug Fix Confirmed

**Original failure**: `handleImport` sent `grade_code: emp.grade` (raw Excel value, e.g. `STEP_1B`) to `create_employee`. Workspace with no matching grade entry returned `400 "Grade 'STEP_1B' not found"` ‚Üí all 180 uploads failed.

**Fix applied**: `grade_code: null` always in the upload path. Grade is a payroll setup field ‚Äî it is assigned exclusively through the Enroll flow, not during upload. The `EmployeeUpload` component retains the grade column for the admin's reference (mapping panel, preview table) but it is no longer sent to the API during bulk import.

**Designation handling**: `designation_code: emp.designation_unresolved ? null : emp.designation || null` ‚Äî resolved designations are sent (designation is HR data); unresolved are silently dropped rather than blocking the import.

## Security Hardening Applied

`bulk_enroll_employee_contracts` ‚Äî both `already_rows` SELECT and the bulk UPDATE now join through `employee.workspace_id` for defence-in-depth workspace scoping on `employee_contract` queries (Sprint 11 Retro).
