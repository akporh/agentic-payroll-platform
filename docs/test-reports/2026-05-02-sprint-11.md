## New Bug Found and Fixed This Session
**derivation_error regression**: `derive_salary_components` called in route's employee loop (before batch_processor's try/except) caused `decimal.InvalidOperation` for employees with non-numeric salary amounts (e.g. test fixture `{"amount": "INVALID"}`). Previously the InvalidOperation propagated from the executor's `calculate_gross()` inside the error boundary — now it fires outside it.

**Fix applied**: Route catches exception, stores `derivation_error` string on emp dict. Batch_processor re-raises it inside the per-employee try block, producing a FAILED result as before.
